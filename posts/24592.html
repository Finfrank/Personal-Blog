A.无穷级数的和函数

题目描述:

定义域为 $N∗$ 的函数 $f(x)=−(x−a)(x−b)$,记$S(i)=\sum_{i=1}^n{​f(x)}=f(1)+f(2)+⋯+f(i)$:

求 $m,n∈(0,+∞)$ 且 $m<n$ 时，$S(n)−S(m)$ 的最大值。

输入:

全文第一行输入一个正整数 $T(1\le T\le10^5)T(1≤T≤105)$，表示数据组数。

接下来 $T$ 行，每行两个正整数 $a,b(1\le a<b\le10^9)$ 描述函数 $f(x)$。

输出:

求所有的整数 $m,n(0<m<n)$ 中，使得 $S(n)-S(m)$ 取得最大时对应

$S(n)-S(m)$ 的值，答案对 998244353 取模，每组输出之间需要换行。

示例 1

3
2 4
2 4
2 4

1
1
1

一组合法的 $m,n$ 值为 $1,4$，此时$f(x)=-x^2+6x-8,S(n)-S(m)=1$

题解

$f(x)$ 的图像大致如图所示

得出$S(m)-S(n)$ 的最大值其实是 $|S(b)-S(a)|$

将$f(x)展开带入S(n)$得 $S(n)= \sum_{i=1}^{n}f(x)=-\sum_{i=1}^{n} x^{2} + (a+b)\sum_{i=1}^{n} x-a\ast b \sum_{i=1}^{n}1$

利用级数求和解得 $S(n)=-(n\ast(n+1)\ast(2\ast n+1))\ / \ 6 + （a+b)\ast (n\ast(n+1))\ / \ 2 + n * a * b$

  • 法一：将公式带入求解,可利用求逆元。

  • 法二：试根法简化，预测原式是一个三项式，令$d=b-a$代入得答案为$d*(d+1)*(d+2) \ / \ 6 % mod$

关于试根法：若有整系数多项式$a_nx^n+……+a_1x+a_0$
则记$f(x)=a_nx^n+……+a_1x+a_0$
分别列出最高次项系数$a_n$的约数和常数项$a_0$的约数，把这些数分别相除，就能得到$f(x)=0$可能的根，代入$f(x)$检验，若$f(a)=0$，则最后多项式必含有因式$(x-a)$，再用综合除法得到剩下的因式

代码

#include <bits/stdc++.h>
using namespace std;
#define ll long long
const ll mod=998244353;
//逆元 此处也可用快速幂
const ll iv2=(mod+1)/2ll
const ll iv6=(mod+1)/6ll
ll S(ll n){
    ll ans1=n*(n+1ll)%mod*(2ll*n+1ll)%mod*iv6%mod;
    ll ans2=(a+b)*(n+1ll)%mod*n%mod*iv2%mod;
    ll ans3=a*b%mod*n%mod;
    ll ans=(-ans1+ans2-ans3);
    ans=(ans%mod+mod)%mod;
    return ans;
}
ll a,b;
int main()
{
    int n;
    cin>>n;
    while(n--)
    {
        cin>>a>>b;
        cout<<((S(a)-S(b)+mod)%mod)<<endl;
    }
    return 0;
}

//利用试根法简化后
#include <bits/stdc++.h>
using namespace std;
#define ll long long
const ll mod=998244353;
ll a,b,c;

int main()
{
    cin>>c;
    while(c--)
    {
        cin>>a>>b;
        __int128 d=b-a-1;
        d = d*(d+1)*(d+2) / 6 % mod;
        cout<<(int)d<<endl;
    }
    return 0;
}

n=(int)(input())
for i in range(0,n):
    a,b=map(int,input().split())
    r=b-a
    print(((r*r*r-r)//6)%998244353)

B.Rabbit Number

题目描述:

What a cute rabbit! Mr.Gree has a lot of cute rabbits in the farm, and every rabbit has a number a1​,a2​,a3​…an​. Different rabbits may have same numbers. Mr.Gree thinks The Cute Value of these rabbits is $a_1​∗a_2​∗a_3​∗…∗a_n​$. One day, he wants to make The Cute Value changed, so he takes away exactly one rabbit and The Cute Value is changed.

Mr.Gree wants to know how many different numbers The Cute Value could be?

For example, Mr.Gree has three rabbits numbered [3,4,5], now The Cute Value is $3∗4∗5=60$. If he takes away a rabbit numbered 3, The Cute Value will change to $4∗5=20$. And if he takes away a rabbit numbered 4, now The Cute Value will change to $3∗5=15$. And if he take away a rabbit numbered 5, now The Cute Value will change to $3∗4=12$. So The Cute Value could change to 12, 15 or 20 three numbers after one rabbit is took away.

输入:

The 1st line of input contains a single integer $n(2\leq n\leq 10^5)$

The 2rd line of input contains n integer $a_i(-10^9 \leq a_i \leq 10^9)$ means the number that the $i−th$ rabbit has.

输出:

print one line contains an integer.

示例1

3
3 4 5

3

示例2

4
1 1 2 2

1

示例3

4
-1 -1 1 1

1

题解

题意：有n个数取出一个数算剩余数的积（即可爱值），求不同的积有多少个？

思路：分类讨论

  • 若 0 的个数大于等于 2，则移除任何数字都对可爱值没有影响。
  • 若 0 的个数为 1，则只有一种方案，即移除 0。（移除其他数字的话，可爱值依旧为 0，不改变）
  • 若没有零，则统计不同数字的出现次数，注意：移除数字为 1 的兔子对可爱值没有影响

代码

#include<bits/stdc++.h>
using namespace std;
long long n;
long long x,flag;
set<long long>sets;
int main()
{
    cin>>n;
    for(int i=1;i<=n;i++)
    {
        cin>>x;
        if(x==0) flag=1;
        if(x!=1)
        sets.insert(x);
    }
    if(flag)cout<<0;
    else cout<<sets.size();
}

n  = int(input())
nums = list(map(int, input().split()))
if nums.count(0) == 1:
    print(1)
    exit(0)
elif nums.count(0) > 1:
    print(0)
    exit(0)
if nums.count(1) > 0:
    print(len(list(set(nums))) - 1)
    exit(0)
print(len(list(set(nums))))

C.最优分割

题目描述:

Apojacsleam现在想邀请他的一些好友去校外吃一顿饭，并在某酒店包下了一桌饭。当Apojacsleam和他的同学们来到酒店之后，他才发现了这些同学们其实是$N$对cp，由于要保护广大单身狗的弱小心灵，所以他不想让任意一对情侣相邻。

说明：

  • 酒店的桌子是恰好有$2N$个位置的圆桌。
  • 客人恰好是$N$对cp，也就是说，圆桌上没有空位。
  • 桌子的每一个位置是一样的，也就是说，如果两种方案可以通过旋转得到，那么这就可以视为相等的。

现在，你需要求出，将任意一对情侣不相邻的方案数。

输入:

一行一个正整数$N$，表示cp的对数。

输出:

一行一个非负整数，表示答案对$1000000007$取模后的值。

对于20%的数据，1<=N<=5
对于30%的数据，1<=N<=20
对于50%的数据，1<=N<=100
对于70%的数据，1<=N<=200000
对于100%的数据，1<=N<=30000000

示例1

2

2

说明：两种方案：假设1-2、3-4是两对情侣。方案有1-3-2-4，1-4-2-3或者1-3-2-4，2-3-1-4均合法

示例2

25

535659175

题解

题意

存在$n$对情侣，一共$2*n$个人。要求每对情侣都不能相邻而坐，问有多少种排序方案？

法一：动态规划

将所有人看成一个环，则1234与2341为同种方案。可以先将$n$个男生挑出来，共$(n-1)!$种方案。然后将剩余$n$个人插入环。

定义$dp[i]$表示安排完前$i$个人的方案数

  • $dp[i-1]*(n+i-3)$
    直接插入。第$i$个人插入时，共有$n+i-1$个人，此时有$n-i+1$个位置。但要减去情侣中另一个人相邻左右2个位置，即$n+i-3$
  • $dp[i-1]*(n+i-3)$
    间接插入。将第$i-1$个人暂时与他的情侣安排在一起，把第$i$个人插入他们中间。

两种方法独立且无重复，需特判$n=1$时的情况

得$dp[i]=dp[i-1] * (n+i-3) + dp[i-1] * (n+i-3)$

最终答案为$dp[n]+(n-1)!$

代码
#include <bits/stdc++.h>
using namespace std;
typedef long long ll;
const int N =3e7+7;
const int MOD =1e9+7;
ll a[N];
int main() 
{
    int n;
    cin>>n;
    if(n==1) return puts("0"),0; //特判
    a[0]=1,a[1]=n-2; //预处理
    for(int i=2;i<=n;i++)
        a[i] = (a[i-1]*(n+i-3) %MOD + a[i-2]*(i-1)*2 %MOD) % MOD; //插入
    for (int i=2;i<n;i++)
        a[n] = a[n] * i % MOD; //计算(n-1)!
    cout<<a[n];
    return 0;
}

法二：组合数+简单容斥

关于容斥原理：
要计算几个集合并集的大小，我们要先将所有单个集合的大小计算出来，然后减去所有两个集合相交的部分，再加回有三个集合相交的部分，再减去所有四个集合相交的部分….依此类推，一直计算到所有集合相交的部分。
设 U 中元素有 n 种不同的属性，而第 i 种属性称为 $P_i$，拥有属性 $P_i$ 的元素构成集合 $S_i$，那么
$$
\begin{split}
\left|\bigcup_{i=1}^{n}S_i\right|=&\sum_{i}|S_i|-\sum_{i<j}|S_i\cap S_j|+\sum_{i<j<k}|S_i\cap S_j\cap S_k|-\cdots\
& +(-1)^{m-1}\sum_{a_i<a_{i+1} }\left|\bigcap_{i=1}^{m}S_{a_i}\right|+\cdots+(-1)^{n-1}|S_1\cap\cdots\cap S_n|
\end{split}
$$
即
$$
\left|\bigcup_{i=1}^{n}S_i\right|=\sum_{m=1}^n(-1)^{m-1}\sum_{a_i<a_{i+1} }\left|\bigcap_{i=1}^mS_{a_i}\right|
$$

根据容斥原理

$ans$ = 全部方案 - 一对相邻 - 两对相邻…

  • 全部方案 $(2n-1)!$

  • 一对相邻 $2 * (2n-2)!$ ，一共有${n\choose 1}$种

  • 两对相邻 $2^2 * (2n-3)!$ ，一共有${n\choose 2}$种

因此 $ans = \sum_{i=0}^n(-1)^i * 2^i * (2n-i-1)! * {n\choose i} $

将${n\choose i}$进行预处理，时间复杂度为$O(n)$

代码
#include <bits/stdc++.h>
using namespace std;
const int N = 3e7 + 5;
typedef long long ll;
const int mod = 1e9 + 7;
int fact[N], infact[N];
//快速幂
ll ksm(ll a, ll b) {
    ll ans = 1;
    while(b)
    {
        if(b&1) ans = ans*a%mod;
        a = a*a%mod;
        b>>=1;
    }
    return ans;
}
//初始化公式
void init(int n) 
{
    fact[0] = infact[0] = 1;
    for (int i = 1; i <= n; i++ )
    {
        fact[i] = (ll)i * fact[i - 1] % mod;
    }
    infact[n] = ksm(fact[n], mod - 2);
    for (int i = n - 1; i >= 0; i-- ) 
    {
        infact[i] = (ll)infact[i + 1] * (i + 1) % mod;
    }
}
//组合数
ll C(ll a,ll b) 
{
    if(a<b)  return 0;
    return (ll)fact[a] * infact[a - b]%mod * infact[b] % mod;
}
int main() 
{
    int n;
    cin>>n;
    if(n <= 1) return puts("0"),0; 
    init(n);
    ll res = 0;
    ll P = 1ll * fact[n-1];
    ll ksm2 = ksm(2, n);
    ll inv2 = ksm(2, mod-2);
    for ( int i = n; i >= 0; -- i  ) 
    {
        if(i&1) res = (res - C(n, i) * ksm2 % mod * P%mod + mod) % mod;
        else res = (res + C(n, i) * ksm2 % mod * P%mod + mod) % mod;
        P = P * (2*n-i)%mod;
        ksm2 = (ksm2 * inv2) % mod;
    }
    cout<<res;
    return 0;
}

D.立体几何与空间变换

题目描述：

小渊是个聪明的孩子，他经常会给周围的小朋友们将写自己认为有趣的内容。最近，他准备给小朋友们讲解立体图，请你帮他画出立体图。

小渊有一块面积为 m×n 的矩形区域，上面有 m×n 个边长为 1 的格子，每个格子上堆了一些同样大小的积木（积木的长宽高都是 1），小渊想请你打印出这些格子的立体图。

我们定义每个积木为如下格式，并且不会做任何翻转旋转，只会严格以这一种形式摆放：

每个顶点用 1 个加号 + 表示，长用 3 个 − 表示，宽用 1 个 / 表示，高用两个 | 表示。

字符 +,−,/,| 的 ASCII 码分别为 43，45，47，124。

字符 . (ASCII 码 46) 需要作为背景输出，即立体图里的空白部分需要用 . 来代替。

立体图的画法如下面的规则：

若两块积木左右相邻，图示为：

若两块积木上下相邻，图示为：

若两块积木前后相邻，图示为：

立体图中，定义位于第 $(m,1)$的格子（即第 m 行第 1 列的格子）上面自底向上的第一块积木（即最下面的一块积木）的左下角顶点为整张图最左下角的点。

输入

输入文件第一行有用空格隔开的两个整数 $m 和 n$，表示有 $m×n $个格子。 

接下来的 $m $行，是一个 $m×n $的矩阵，每行有$ n $个用空格隔开的整数，其中第 $i $行第$ j $列上的整数表示第 $i$ 行第 $j$ 列的格子上摞有多少个积木$（1≤每个格子上的积木数≤100）$。

输出

输出文件中包含题目要求的立体图，是一个 K 行 L 列的字符矩阵，其中 K 和 L 表示最少需要 K 行 L 列才能按规定输出立体图。

$1≤m,n≤50$

题解

思路：字符串处理 模拟 坐标变换

首先定义小正方体

char box[6][8] = {
    "..+---+",
    "./   /|",
    "+---+ |",
    "|   | +",
    "|   |/.",
    "+---+.."
};

按照后-前、左-右、下-上的顺序画小正方体

将最靠左、前、下的小正方体和二维坐标系的$(499,0)$对齐，由投影得出坐标的规律：

设小正方体在$(x,y,z)$,横纵坐标变换$X Y$

  • 横坐标：$\bigtriangleup z = 1, \bigtriangleup X = 3;\bigtriangleup x = 1, \bigtriangleup Y = 2$

  • 纵坐标：$\bigtriangleup x = 1, \bigtriangleup 总 = 3;\bigtriangleup y = 1, \bigtriangleup Y = 2$

得出投影后左下角的坐标为$((499-3z-2(n-1-x)),2(n-1-x)+4y)$

最后输出二位投影平面内的部分 (要注意ASCII 码规范)

时间复杂度 ： $O(42*nmh)$

代码

#include <bits/stdc++.h>
using namespace std;
int n,m;
char box[6][8] = {
    "..+---+",
    "./   /|",
    "+---+ |",
    "|   | +",
    "|   |/.",
    "+---+.."
};
char g[500][500];
int h[500][500];
int i,j,x,y,z;
int up=500,right=0;
int X,Y,a,b;
int main()
{
    cin>>n>>m;
    for(i=0;i<n;i++)
    for(j=0;j<m;j++) 
    cin>>h[i][j];

    for(i=0;i<500;i++)
    {
        for(j=0;j<500;j++)
        g[i][j]='.';
    }

    for(x=0;x<n;x++)
        for(y=0;y<m;y++)
            for(z=0;z<h[x][y];z++)
            {
                X=499-2*(n-1-x)-3*z,Y=2*(n-1-x)+4*y;
                up=min(up,X-5),right=max(right,Y+6);
                for(a=0;a<6;a++)
                for(b=0;b<7;b++)
                if(box[a][b]!='.')
                g[X-5+a][Y+b]=box[a][b];
            }

    for(i=up;i<500;i++)
    {
        for(j=0;j<=right;j++) 
        cout<<g[i][j];
        cout<<endl;
    }
    return 0;
}

E.矩阵搜索

题目描述:

Lancer has a $01$ matrix with $M$ rows and $N$ columns (matrix containing only numbers $0 or 1$), perform $Q$ times of queries, but he can only remember a small matrix each time. Can you help him to how many matrix has appeared in the original matrix ?

输入

The first line contains four integer $M,N,A,B (A≤100，M,N,B≤1000，Q≤1000)$

The following $M$ rows and $N$ columns is the original matrix.

Then,descriptions of $Q$ test cases follow.

Each cases gives an $01$ matrix with $ A$ rows and $ B$ columns.

(There’s no space between numbers)

输出

Output a integer on a line - the sum of the small matrix that has appearced.

示例1

3 3 2 2
111
000
111
3
11
00
11
11
00
11

2

题解

主要思路 ： 二维hash

对于原矩阵中所有大小为AxB的子矩阵求出其Hash值并储存，每次输入一个小矩阵计算对应的Hash来查找。

暴力求解复杂度为$O(nmab)$，无法通过。需要进行优化

一维哈希中我们按照一个“方向”计算，通过自左向右、自上而下的”方向”实现二维的转换。

预处理$f[i][j]$表示第i行中，1~j的Hash值。由Hash函数的可减性，将输入的子矩阵每一行的Hash值的时间复杂度从$O(b)$优化为$O(1)$，所以总时间复杂度优化到$O(nma)$。

#include <bits/stdc++.h>
using namespace std;
typedef unsigned long long ULL;
const int N=1010,M=N*N,P=131;
ULL h[N][N],p[M];
int n,m,a,b;
int sum=0;
char str[N];
ULL calc(ULL f[],int l,int r)
{
    return f[r]-f[l-1]*p[r-l+1];
}
int main()
{
    cin>>n>>m>>a>>b;
    p[0]=1;
    for(int i=1;i<=n*m;i++)
        p[i]=p[i-1]*P;

    for(int i=1;i<=n;i++)
    {
       cin>>(str+1);
       for(int j=1;j<=m;j++)
           h[i][j]=h[i][j-1]*P+str[j]-'0';
    }
    unordered_set<ULL>S;
    for(int i=b;i<=m;i++)
    {
        ULL s=0;
        int l=i-b+1,r=i;
        for(int j=1;j<=n;j++)
        {
            s=s*p[b]+calc(h[j],l,r);
            if(j>a)s-=calc(h[j-a],l,r)*p[a*b];
            if(j>=a)S.insert(s);
        }
    }
    int Q;
    cin>>Q;
    while(Q--)
    {
        ULL s=0;
        for(int i=1;i<=a;i++)
        {
            cin>>str;
            for(int j=0;j<b;j++)
            s=s*P+str[j]-'0';
        }
        if(S.count(s)) sum++;
    }
    cout<<sum;
    return 0;
}
